# Requirements

## Functional Requirements
1. Users can create, view, edit, and close support tickets.
2. Tickets can be assigned to agents or queues.
3. Agents can add internal notes and public replies.
4. Users can manage customer profiles and case history.
5. The system should support ticket priorities and SLA tracking.
6. Managers can review ticket volume and response performance.

## Non-Functional Requirements
- Fast page loads for daily support operations
- Secure authentication and role-based access control
- Reliable storage for long-term auditability
- Clear error handling and log visibility
- Mobile-friendly UI for remote teams

## Assumptions
- The product will start with a single-tenant deployment model.
- Email integration can be added after core ticketing is stable.
